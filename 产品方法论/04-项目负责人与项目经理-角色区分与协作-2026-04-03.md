# 项目负责人与项目经理：角色区分与协作模型

> 来源：项目负责人与项目经理的核心区别解析
> 提炼日期：2026-04-03
> 分类：产品方法论

## 核心观点

1. **项目负责人关注"做正确的事"，项目经理关注"正确地做事"**。前者聚焦"做什么"与"为什么做"（战略方向+业务价值），后者聚焦"如何做"与"何时完成"（执行计划+交付管控）。

2. **在敏捷环境中，项目负责人 ≈ PO（产品负责人），项目经理 ≈ Scrum Master**。一个最大化产品价值，一个保障团队高效运作。

## 关键知识点

### 多维度对比速查表

| 维度 | 项目负责人 | 项目经理 |
|------|-----------|---------|
| 核心问题 | 做什么？为什么做？ | 如何做？何时完成？ |
| 时间视角 | 长期，产品全生命周期 | 短期，特定项目执行周期 |
| 对团队 | 定义需求、明确优先级 | 分配任务、跟踪进度 |
| 变更管理 | 调整产品方向 | 管控范围蔓延 |
| 权责 | 战略决策权 | 执行管理权（无战略决策权） |
| 汇报 | 向高层/业务方 | 向项目负责人 |
| 产出物 | 愿景、需求文档、优先级清单 | 项目计划、进度报表、风险报告 |
| 风险关注 | 市场风险、需求风险 | 进度风险、成本风险 |
| 成功标准 | 市场认可度、用户满意度 | 按时按预算按范围交付 |
| 典型背景 | 业务/市场/领域专家 | 工程管理/PMP 认证 |

### 核心技能差异

**项目负责人**：领域知识 + 市场洞察 + 战略决策 + 干系人管理 + 价值评估

**项目经理**：项目规划 + 风险管理 + 团队领导 + 跨部门协调 + 预算资源管控

### 协作模型

```
项目负责人 ──(方向+优先级+价值导向)──> 项目经理
项目经理 ──(执行状态+风险预警+交付进展)──> 项目负责人
```

## 启发与思考

1. **面试定位参考**：如果面试的是"产品经理"岗位，实际上更偏向"项目负责人"的角色——核心是业务价值判断和需求优先级决策，而非执行层面的进度管控。沃丰科技 AI 客服 PM 岗位，应该强调自己的业务洞察力和价值评估能力。

2. **与之前笔记的关联**：01 号笔记（项目经理三大陷阱）讲的是项目经理角色的反模式；本篇补充了项目负责人的视角。两者结合，可以更完整地理解"产品经理"在实际工作中的双重角色。

3. **AI 时代的变化**：结合 03 号笔记（PRD 已死原型万岁），项目负责人的产出物正在从"需求文档"变成"可交互原型"，而项目经理的角色可能被 AI 工具部分替代（进度追踪、风险识别等）。

## 原文精华

> 项目负责人聚焦于"做什么"与"为什么做"，立足业务本质把控项目整体方向；项目经理专注于"如何做"与"何时完成"，聚焦执行过程保障项目落地实施。

> 项目负责人的核心职责是确保"做正确的事"；项目经理的核心职责是确保"正确地做事"。
