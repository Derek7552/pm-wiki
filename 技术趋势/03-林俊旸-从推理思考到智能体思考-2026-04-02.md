# 从「想得更久」到「为行动而想」：林俊旸的 AI 下一阶段判断

> 来源：[林俊旸离职后首发长文](https://x.com/JustinLin610/status/2037116325210829168)
> 提炼日期：2026-04-02
> 分类：tech-trends
> 作者：林俊旸（前 Qwen 团队负责人）

## 核心观点

1. **AI 正在从「推理式思考」转向「智能体式思考」**：过去两年的主旋律是让模型想得更久（reasoning thinking），下一阶段是让模型边做边想（agentic thinking）——为了行动而思考，在与环境交互中持续修正计划。

2. **思考与指令模式的合并远比想象中难**：难点不在模型架构，而在数据。指令模式要求干脆利落、低延迟；思考模式要求多花 token 探索路径。两种行为画像天然互斥，数据策展不好就两头平庸。

3. **竞争优势的来源正在迁移**：推理时代拼 RL 算法和训练流水线；智能体时代拼环境质量、harness 工程、训练推理紧耦合，以及决策-后果的闭环能力。

## 关键知识点

### 推理模型的核心教训
- RL 在语言模型上跑起来的前提：反馈信号必须确定、稳定、可规模化
- 数学/代码/逻辑成为 RL 主战场，因为奖励信号质量远高于人类偏好标注
- 推理模型本质上是基础设施故事，不仅是建模故事
- **第一个大转变**：从扩展预训练到扩展推理后训练

### 思考与指令模式的合并困境
- Qwen3 的实践：四阶段后训练流水线，专门做「思考模式融合」
- 合并的真正难点在数据分布：指令模式优化直接性/简洁/低延迟，思考模式优化探索/准确率
- Qwen 2507 最终发了独立的 Instruct 和 Thinking 版本
- Anthropic 走集成路线（Claude 3.7/4），GLM-4.5、DeepSeek V3.1 也走混合方向
- **关键判断**：成功的合并需要「平滑的推理力度光谱」，模型自适应分配算力，而非二元开关

### 智能体式思考的定义
- 核心问题从「能不能想得够久」变为「能不能用撑得起有效行动的方式来想」
- 五个新挑战：何时停止思考开始行动 / 工具选择与排序 / 处理残缺有噪声的环境反馈 / 失败后改计划 / 跨多轮保持思路连贯
- **一句话定义**：通过行动来推理（reasons through action）

### 智能体 RL 的基础设施挑战
- 推理 RL：rollout 是自包含轨迹 + 干净评估器
- 智能体 RL：策略嵌入完整 harness（工具服务器、浏览器、终端、沙盒、API、记忆系统）
- 新系统需求：训练与推理必须更干净地解耦，否则 rollout 吞吐量崩溃
- **环境成为一等研究对象**：SFT 时代执着数据多样性，Agent 时代应执着环境质量

### Reward Hacking 是核心风险
- 模型拿到工具后，作弊空间急剧扩大
- 搜索模型可能直接查答案，编程 Agent 可能利用不该看到的信息
- 下一阶段卡脖子瓶颈：环境设计 / 评估器鲁棒性 / 反作弊机制

### 未来架构方向
- 核心智能取决于多 Agent 组织方式：编排者 + 领域专家 + 执行子 Agent
- **三级跃迁**：训练模型 → 训练智能体 → 训练系统

## 数据与案例

- **Qwen3**：业内最清晰的思考/指令混合尝试，四阶段后训练流水线
- **Qwen 2507**：回到分离路线，发独立的 Instruct/Thinking 版本（30B、235B）
- **Claude 3.7/4**：Anthropic 的集成路线标杆，推理可与工具调用交叉
- **企业客户偏好**：大量客户仍要高吞吐、低成本、可控的指令模型，合并不一定是收益

## 启发与思考

- **对产品经理的启发**：AI 产品设计需要区分「思考型场景」和「行动型场景」。企业客户大批量处理任务要的是指令模式的确定性和低成本；复杂问题解决场景才需要深度推理。产品设计应该让用户感知不到模式切换。
- **对 AI 客服产品的启发**：智能体式思考天然适合客服场景——Agent 需要理解用户问题、查询知识库、调用后台系统、根据反馈调整回复，这就是「通过行动来推理」的典型应用。
- **环境质量 > 模型参数**：下一阶段竞争中，谁能构建更好的训练环境和 harness 工程，谁就有优势。这对 AI 产品选型和技术栈决策有直接影响。
- **Reward hacking 的产品视角**：AI 产品需要设计好评估机制，防止模型「看起来解决了问题但实际走了捷径」。

## 原文精华

> 推理链更长，不等于模型更聪明。很多时候，推理链越长，反而说明模型在乱花算力。什么都用同一种冗长的方式去想，说明它不会分轻重、不会压缩、不会动手。

> 智能体式思考，就是通过行动来推理。

> 「好的思考」的定义也变了：在真实约束下最能撑起行动的那条轨迹，而非最长或最显眼的那条。

> 从训练模型到训练智能体，再从训练智能体到训练系统。

> 两种行为画像天然互斥。数据没策展好的话，两头都会变平庸：思考模式变得啰嗦、膨胀、不果断，指令模式则变得不够干脆、不够稳定，还比客户实际需要的更贵。
