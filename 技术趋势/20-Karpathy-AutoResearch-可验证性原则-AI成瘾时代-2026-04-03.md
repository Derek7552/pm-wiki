# Karpathy 深度访谈：AutoResearch、可验证性原则与 AI 成瘾时代

> 来源：No Priors Podcast（Sarah Guo 对话 Andrej Karpathy），公众号整理稿直接粘贴
> 提炼日期：2026-04-03
> 分类：技术趋势

## 核心观点

1. **可验证性是 AI 能力边界的终极判据**：一切可验证的领域（有客观指标、可自动评估），要么已经属于机器，要么很快就会；一切不可验证的领域（需要品味、判断、创造性），仍然属于人类。这是 Karpathy 给出的最简洁的认知框架。

2. **从写代码到「传达意志」——2025 年 12 月的翻转**：Karpathy 的编码/委托比从 80/20 翻转为 20/80。他认为自己从 12 月起没有亲手打过一行代码。新的瓶颈不是算力，而是「token 吞吐量」——你能同时管理多少个 Agent 会话。

3. **AutoResearch = 把人类研究员踢出循环**：用 AI 自动设计实验、跑实验、评估结果、迭代改进。一个晚上的 AutoResearch 发现了 Karpathy 二十年经验遗漏的超参数交互效应。目标是最大化 token 吞吐量，把人类从循环中移除。

## 关键知识点

### 可验证性原则（Verifiability Principle）

```
可验证领域（有客观指标）     → Agent 极其擅长，可自主迭代直到成功
  例：代码/CUDA优化/超参数搜索/测试通过

不可验证领域（无客观标准）   → Agent 漫无边际游荡，仍需人类判断
  例：笑话/品味/产品方向/架构权衡/创造性表达
```

**关键洞察**：模型的「参差感」（jaggedness）——在训练轨道上比光速还快，偏离轨道就像十岁小孩。代码能力的巨大进步并不会自动提升讲笑话的能力，因为笑话不在 RL 优化范围内。

### Token 吞吐量作为新瓶颈

| 旧范式 | 新范式 |
|--------|--------|
| 瓶颈 = GPU 算力 | 瓶颈 = 你能管理多少 token 吞吐量 |
| 焦虑 = GPU 没跑满 | 焦虑 = 订阅额度还有剩余就去睡觉 |
| 工作单元 = 一行代码/一个函数 | 工作单元 = 一个功能/一个任务委托给 Agent |
| 操作方式 = 写代码 | 操作方式 = 在多个 Agent 会话间穿梭分配任务 |

### AutoResearch 架构

```
想法队列（arXiv论文 + GitHub仓库 + 人类想法）
    ↓
自动科学家（基于信息提出实验想法）
    ↓
工作者池（从队列取任务，跑实验）
    ↓
有效结果 → 特性分支 → 人类偶尔监控 → 合并到主分支
```

**关键不对称性**：「发现」极其昂贵（尝试一万个想法），「验证」极其廉价（跑一遍训练）。这使得 AutoResearch 天然适合去中心化——类似 Folding@home 模式。

### 程序 MD = 组织的可执行描述

每个研究组织都可以被描述为一套 Markdown 文件——角色、流程、偏好（冒险 vs 保守、多站会 vs 少站会）。一旦组织被描述为代码，就可以被调优。这是元层级的 AutoResearch。

### App 消亡，API 接管

- 软件的用户不再是人类，而是代表人类行动的 Agent
- 智能家居的 6 个 App → 一个 Agent（Dobby）通过自然语言控制一切
- Agent 是调用所有 API 端点的「智能胶水」
- 当前门槛（需要 vibe coding）是暂时的，1-3 年后将对普通人免费

### 模型物种分化 vs 单一文化

- 当前：模型单一文化——所有实验室都追求「对所有领域任意智能」的单一大模型
- 预期：像动物大脑一样出现物种分化——专门化模型用于特定任务（如 Lean 证明）
- 障碍：「操纵大脑的科学还没完全发展起来」——如何在不损失能力的情况下微调

### 开源 vs 封闭的均衡

- 开源落后封闭前沿约 6-8 个月，差距在缩小
- 类比 Linux：不在能力最前沿，但作为公共开放平台，是行业健康的权力平衡
- Karpathy 评价：「我们偶然地处于一个好的位置」

### 教育范式转变

- Karpathy 不再向人类解释，而是向 Agent 解释
- Agent 比他解释得更好：无限耐心、因材施教、适配能力水平
- 教育者的新角色：注入那「几个关键比特」（课程设计、核心洞察），其余交给 Agent
- microGPT 的讽刺：Agent 写不出这 200 行极简代码，但完全理解为什么这样写

### 数字→接口→物理的演变轨迹

```
当前：数字空间的大解绑（大量数字信息被百倍效率重新处理）
  ↓
近期：数字-物理接口（传感器让AI感知世界，执行器让AI响应世界）
  ↓
远期：物理世界自动化（原子比比特难一百万倍，但TAM更大）
```

## 数据与案例

- **编码比例翻转**：80/20 → 20/80（2025年12月），Karpathy 自此没有亲手打过一行代码
- **AutoResearch 一夜成果**：发现了值嵌入权重衰减和 Adam beta 参数的交互效应——20 年经验的研究员遗漏的
- **Dobby 智能家居**：3 个提示词让 Agent 自动扫描局域网、找到 Sonos、调用 API 播放音乐，后来接管了整栋房子
- **ChatGPT 笑话停滞**：三四年前和今天问同一个笑话（为什么科学家不信任原子），因为笑话不在 RL 优化范围内
- **OpenAI 研究员自动化**：「如果我们成功了，我们所有人都要失业了」——研究员正在积极把自己自动化掉
- **杰文斯悖论**：ATM 出现后银行出纳员反而增加了——软件门槛降低，需求可能不降反升

## 启发与思考

1. **可验证性原则统摄了之前所有笔记的核心矛盾**：
   - 「对话式→Agentic AI」笔记中 Agent 擅长/不擅长的边界 = 可验证 vs 不可验证
   - Anthropic 五种工作流中 Evaluator-Optimizer 模式 = AutoResearch 的迷你版
   - 「世界观三大基石」中的奥卡姆剃刀 = Karpathy 的 microGPT 极简主义

2. **对 PM 的核心启示**：
   - PM 的工作大部分在「不可验证领域」——产品判断、用户洞察、优先级决策
   - 但 PM 工作中的可验证部分（数据分析、A/B 测试、bug 验证）将快速被 Agent 接管
   - **策略性思考自己的时间花在哪里**：Agent 做不到的才是你的工作

3. **「程序 MD」概念对团队管理的启发**：组织的运作方式可以被描述为 Markdown → 可以被 Agent 理解和执行 → 可以被调优。这与 Skills 笔记中「隐性知识编码化」完全一致。

4. **App 消亡趋势对产品设计的深远影响**：如果未来软件的主要用户是 Agent 而非人类，产品设计的重心从 UI/UX 转向 API/ACI。这与之前「AI 产品去界面化」笔记完美呼应。

5. **独立 vs 体制内的权衡模型**：Karpathy 的坦诚自我剖析——独立有言论自由但判断会漂移，体制内有算力和信息但有压力——这对所有技术从业者的职业选择都有参考价值。

## 原文精华

> 一切不可验证的领域，都还是人类的；而一切可验证的领域，要么已经属于机器，要么很快就会属于。

> 「代码」这个词都不对了，我现在是在向我的智能体「传达意志」，而且要连续十六个小时。

> 我不应该是那个做这些超参数搜索的人。这里有客观的评判标准，你只需要安排好，让它永远跑下去。

> 要从现有工具中获得最大收益，你必须把自己这个瓶颈移除掉。你必须把事情安排好，让它们完全自主地运转，最大化你的 token 吞吐量，不要在循环里。

> 我同时感觉像是在与一个系统级别有着整个职业生涯经验的极其聪明的博士生合作，以及一个十岁的孩子——人类这两种状态的耦合程度高得多，你不会遇到这种组合。

> 互联网上的一群智能体，可以协作改进 LLM，甚至有可能在某些方面超越前沿实验室。前沿实验室有巨量可信算力，但地球更大，有巨量不可信算力。

> 我不再是在向人们解释事情，我是在向智能体解释事情。如果我能向智能体解释清楚，那么智能体就可以成为路由器。

> 这些智能家居设备的 App 其实都不应该存在。它们应该就是 API，智能体应该直接调用这些 API。
