# 所有 Agent 本质上都是 Coding Agent：SaaS 颠覆、Proactive Agent 与「AI 管 AI」

> 来源：42章经播客（曲凯对话文锋），公众号整理稿直接粘贴
> 提炼日期：2026-04-03
> 分类：技术趋势

## 核心观点

1. **所有 Agent 本质上都是 Coding Agent**：OpenClaw 的核心是一个叫 Pi 的 Coding Agent，其余都是套壳。Coding Agent + Skill 基本可以覆盖大多数垂直场景，垂直 Agent 这条路需要被高度怀疑。今年的机会是谁能把 Coding Agent 的「套壳」做得足够好。

2. **SaaS 被双重颠覆——软件价值被 AI Coding 替代，know-how 价值被 Skill 替代**：SaaS 本质是标准化 SOP，让普通人获得接近专家的能力。但 Agent 让获得专家能力变得更容易——直接说目标，它自己提方案、解决问题、自我迭代。AI 相当于给每家公司配了一个客制化专家，而非通用 best practice。

3. **从「人管 AI」到「AI 管 AI」**：Manus 证明了 Agent 端到端完成任务（人管 AI）；OpenClaw 让大家看到了 AI 管 AI 的可能性。Proactive Agent 的终极形态是能管理其他 AI 的 Agent——自己提需求、搭建临时 Agent、完成后沉淀经验并释放。

## 关键知识点

### Coding Agent 三个阶段

| 阶段 | 代表产品 | 交互模式 | 人的角色 |
|------|---------|---------|---------|
| 第一阶段 | Copilot | 代码补全 | 写代码的人 |
| 第二阶段 | Claude Code | 主动 prompt | AI 指挥者 |
| 第三阶段 | AI 管 AI | AI 自己发现需求、调度执行 | 质检员 |

### 长程任务的突破：从状态机到文件系统

| 去年 | 今年 |
|------|------|
| Agent = 状态机，状态存内存，不可逆 | 状态落到文件上，做错了可以回溯修复 |
| 人控制 context | Agent 自己维护 context |
| 步骤相乘导致准确率崩塌 | 文件系统让错误可见、可修复 |
| 几十步任务 | 几百步甚至上千步任务 |

核心转变：**Less structure, more intelligence**——最好的 context 就是给模型一台电脑或一个文件夹。

### Proactive Agent 的两层主动性

1. **被动触发的主动**：定时任务（每天 8 点发工作总结）——人定义触发条件
2. **真正的 Proactive**：基于过去交互，主动判断用户需要什么，主动提供方案——AI 定义触发条件

关键前提：**context, not control**——给足 context，Agent 足够懂你，就会主动处理。

### Agent 时代的新分层逻辑

- 不按组织人数分层，而是**按 Token 消耗分层**
- 日消耗 1 亿 Token 的用户 ≈ SaaS 时代的世界 500 强
- 一个人/小组织一年贡献 10 万美金完全可能（= 原来招工程师的预算）

### 创业策略转变：从预判到跟随

| 去年 | 今年 |
|------|------|
| 解决 5-10 年后的问题 | 只看未来 3-6 个月 |
| 预判为主 | 预判为辅，跟随为主 |
| 优化赔率（差异化故事） | 优化胜率（确定性高的事情先做好） |
| 追求 100% 准确等远期目标 | 解决大家已经遇到的瓶颈 |

**反思**：不要去解决那些人们还没遇到瓶颈的问题。预判有效期越来越短，从半年缩短到一两个月。

### 人不可被替代的两个点
1. **Taste**（品味/判断力）
2. **协作默契**：点一下就知道在想什么——来自长期合作或之前的 context（如在字节待过）
- 如果需要掰开揉碎讲清楚才能理解，AI 在这个时间内早就做完了

## 数据与案例

- **OpenClaw 作者单日 1600 次 commit**：≈ 三四人团队一年工作量
- **Cursor 一周做出浏览器**：300 万行代码（2025年1月）
- **Anthropic 用 Agent 端到端实现 C 语言编译器**
- **Claude Code 50%+ 使用场景不是 Coding**：数据分析、marketing、文案
- **Agent 用户渗透率 < 1%**：Claude Code + Codex ≈ 500 万活跃用户，未来 vision 是 10 亿
- **效率提升 10 倍**：过去一年的产品，现在两周就够
- **7 人团队 ≈ 过去 30-50 人产出**
- **使用水平差距巨大**：月消耗 2-3 万美元的 90 分用户 vs 大多数人 10 分水平，差距可达 1000 倍

## 启发与思考

1. **「所有 Agent 都是 Coding Agent」统一了之前多篇笔记**：
   - Harness 六层架构 = Coding Agent 的骨架
   - Skills = SaaS know-how 的替代方案
   - Karpathy 的可验证性原则 = Coding Agent 擅长的领域（代码可验证）

2. **对沃丰科技的直接冲击**：如果 SaaS 被双重颠覆，沃丰这类 SaaS 客服产品面临转型压力。但 Anthropic 确认客服是 Agent 最成熟的落地场景——关键是沃丰能否从 SaaS 转型为 Agent Harness 提供商。面试中可以展示这个洞察。

3. **PM 的生存策略**：
   - taste + 协作默契 = PM 不可被替代的两个点
   - 「点一下就知道在想什么」= PM 理解业务的深度
   - 「生产能力越丰饶，真正有意思的东西反而越稀缺」= idea is NOT cheap

4. **「预判为辅，跟随为主」对产品策略的启示**：不要为差异化而差异化（那只是 ego），解决大家已经遇到的瓶颈，顺着明确趋势往前多走半步。

5. **负反馈循环的宏观风险**：如果 AI 替代太多劳动力 → 消费需求下降 → 需求盘子萎缩。这是整个 AI 行业需要面对的结构性问题。

## 原文精华

> 接下来所有 Agent，本质上都是 Coding Agent。

> 去年的逻辑，还是人去控制 context；但今天我们会发现，最有效的方式不是人去控制 context，而是让 Agent 自己去维护 context。

> 生产能力越丰饶，真正有意思的东西反而越稀缺。

> 如果一件事情需要我掰开揉碎讲清楚，一个员工才能理解，那 Ta 可能就比较危险，因为我有和 Ta 解释的这个时间，早就能让 AI 把事情做完了。

> 不要去解决那些人们还没遇到瓶颈的问题。

> 很多时候大家为了讲差异化，会过度放大自己的 ego。

> 我现在只考虑未来 3 到 6 个月的事情，因为我也不知道将来会变成什么样子。
