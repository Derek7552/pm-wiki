## 核心框架

思考质量而非时长决定人与人的差距。思考可分为四个递进层次：

```
混乱 → [拆分] → 清晰 → [归纳] → 规律 → [联系] → 格局 → [提炼] → 本质
```

---

## 四层详解

### 1. 拆分 — 思考的入口

**作用**：把复杂/无从下手的问题切成可处理的小块

**典型应用**：

| 场景 | 拆分维度 |
|------|---------|
| 任务太多 | 紧急 / 重要 / 可延后 |
| 问题太难 | 原因 / 条件 / 步骤 / 风险 |
| 情绪太乱 | 事实 / 感受 / 期待 / 行动 |

**判断标准**：拆到每一块都能独立处理为止。

---

### 2. 归纳 — 思考的沉淀

**作用**：从一堆相似经历/信息中提取共同规律，让经验可复用

**三种归纳**：
- **错误归纳**：同样的错误反复犯 → 找出根因（不是事件本身，是触发条件）
- **模式归纳**：同类事情总发生 → 识别结构性规律
- **���法归纳**：多次成功/失败 → 提炼可迁移方法

**核心价值**：不再从零开始，每次行动都站在已有经验上。

---

### 3. 联系 — 思考的格局

**作用**：打破孤立视角，把单点串成网络

**三种联系**：
- **时间轴联系**：过去 ↔ 现在，看清趋势方向
- **因果联系**：原因 ↔ 结果，提前识别陷阱
- **层级联系**：局部 ↔ 整体，避免钻牛角尖

**判断标准**：从"这件事怎么办"升级到"这��事背后是什么规律"。

---

### 4. 提炼 — 思考的灵魂

**作用**：删繁就简，透过现象抓本质，形成可守住的原则

**三类提炼**：
- 信息 → 一句话结论
- 经历 → 底层逻辑
- 关系 → 相处原则

**最高标准**：复杂的事，一句话说清；重要的理，一个原则守��。

---

## 使用流程

```
遇到问题
  ↓
① 拆分：把问题切成可独立处理的模块
  ↓
② 归纳：找同类经验的共同规律
  ↓
③ 联系：串联上下文/前因后果/局部整体
  ↓
④ 提炼：一句话说清本质结论或原则
```
