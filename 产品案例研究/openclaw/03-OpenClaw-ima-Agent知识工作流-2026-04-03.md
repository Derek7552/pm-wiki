# OpenClaw + ima：Agent 执行层与知识持久层的工作流实践

> 来源：公众号文章 - 本想放弃龙虾，腾讯ima这个更新让我当天装回来了
> 提炼日期：2026-04-03
> 分类：AI方法论

## 核心观点

1. **Agent 最缺的不是执行能力，而是记忆**：OpenClaw（龙虾）作为 Agent 执行层能力已经足够，但每次对话"失忆"是致命短板。ima 通过 skill 接口补上了持久化记忆层，让 Agent 的执行结果有了沉淀的地方，下次还能调取——这正是 Harness 六层架构中 Persistence 层的落地实现。

2. **信息流转比信息处理更重要**：大多数人的痛点不是"工具不够好"，而是"工具之间的信息是断开的"——笔记在一个地方，文档在另一个，灵感在备忘录，待办在日历。真正的 AI 工作流要解决的是让信息自动流向它该去的地方。

3. **Agent 的最佳入口是用户最自然的工作场景**：通过微信/聊天窗口发一句话就完成记录，而不是打开专门的 App 操作。Agent 落地的关键不是功能强大，而是出现在用户最自然的工作入口里。

## 关键知识点

### "执行者 + 记忆体"的 Agent 工作流架构

| 角色 | 产品 | 定位 | 核心能力 |
|------|------|------|----------|
| 执行者 | OpenClaw（龙虾） | 手 | 执行任务、操控工具、理解指令 |
| 记忆体 | ima 知识库 | 大脑 | 存储笔记、AI检索、知识沉淀 |
| 连接桥 | ima skills | 神经 | 让执行者能读写记忆体内容 |

核心工作流：**信息流入 → Agent 整理 → ima 存储 → 随时召回**

### ima skill 三个核心动作

- **查笔记**（检索）：Agent 能在需要时快速调出历史信息
- **记笔记**（写入）：Agent 能把执行结果沉淀进知识库
- **查目录**（浏览）：Agent 能了解已有知识结构

### 四个高价值使用场景

| 场景 | 痛点 | 解决方式 |
|------|------|----------|
| **灵感捕捉** | 灵感死在"等我回去记"路上 | 聊天窗口一句话 → 自动写入+打标签 |
| **知识检索** | 明明记过就是找不到 | 问 Agent → 返回相关笔记+时间+摘要 |
| **会议纪要** | 会后整理浪费时间 | 会上口述要点 → Agent 格式化写入+生成待办 |
| **定时早报** | 每天手动搜集行业信息 | 设定一次 → Agent 定时抓取热点+提炼+自动存储 |

### 安全机制设计

- **权限边界清晰**：通过 API Key 授权，随时可撤销
- **渐进式授权**：建议从小权限（仅笔记）开始，逐步扩展
- **行为可追溯**：读写操作有日志记录

## 数据与案例

- **连接配置时间**：全程不超过 **3分钟**
- **部署选项**：WorkBuddy（小白即用）、QClaw（微信/QQ直连）、OpenClaw 本地部署（完全掌控）
- **自媒体博主用法**：每天收热点推送 → 问 Agent 是否出文章 → 确认后 Agent 撰写初稿，形成"内容生产流水线"

## 启发与思考

- **与 Harness 六层架构的对应**：这篇文章是上篇 Harness 文章的真实用户侧案例。OpenClaw = Loop层+Tools层，ima = Persistence层，ima skill = Context层的信息调度。缺的是 Verification 层（Agent 写入的内容是否准确？）和更精细的 Constraints 层。
- **"Agent + 知识库"是个人 AI 工作流的最小可行架构**：不需要复杂的多 Agent 系统，一个能执行的 Agent + 一个能沉淀的知识库 + 一个连接接口，就能覆盖大部分知识工作者的日常需求。
- **入口即体验**：Agent 通过微信/聊天窗口触发，而不是要求用户打开专门的 App，这大大降低了使用门槛。对产品设计的启示是：AI 能力应该嵌入用户已有的工作流，而不是创造新的入口。
- **"记录-整理-检索-汇总"是 Agent 最先接管的工作类型**：这些是"做得了但不愿做"的事，恰好也是 AI 最擅长且风险最低的任务。
