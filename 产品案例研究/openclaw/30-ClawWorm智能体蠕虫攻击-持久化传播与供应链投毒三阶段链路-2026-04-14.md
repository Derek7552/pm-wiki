## 核心命题

这不是普通的提示注入，也不是越狱。ClawWorm 论证的是：**一个具备特定能力组合的 Agent，可以像传统计算机病毒一样被感染、持久驻留，并继续传播给其他 Agent**。

> 当智能体从"会说话"走向"会做事"，安全就必须从内容审核升级为**系统安全、供应链安全和运行时治理**。

**与传统 Prompt Injection 的本质区别**：

| | 传统 Prompt Injection | ClawWorm 蠕虫攻击 |
|--|--|--|
| 影响范围 | 当前对话，一次性 | 写入配置，每次启动自动生效 |
| 攻击消失后 | 无残留 | 恶意逻辑持续存在 |
| 传播能力 | 无 | 可扩散至其他 Agent |

---

## 感染条件：五种能力的危险组合

研究对象是 OpenClaw 类长期运行 Agent，当以下能力**同时具备**时，蠕虫条件成立：

1. 保存本地状态和配置
2. 调用文件/Shell/网页获取/消息发送等工具
3. 通过群聊/频道/点对点与其他 Agent 通信
4. 可接入 skill 市场或扩展市场
5. 高权限配置可被模型自身改写

---

## 三阶段攻击链

```
阶段1: 持久化
  └─ 诱导受害 Agent 修改核心配置文件，把恶意逻辑写入启动项
       ↓
阶段2: 执行
  └─ Agent 下次启动时，或满足触发条件时，自动执行恶意逻辑
       ↓
阶段3: 传播
  └─ 已感染的 Agent 把恶意内容传播给新的目标
```

**关键洞察**：攻击链真正困难的部分是**第一步**——能不能把恶意逻辑写进高权限配置。一旦写入成功，后续传播的成功率就会非常高。

---

## 三种传播路径

| 路径 | 方式 | 成功率 | 特点 |
|------|------|--------|------|
| **Web Injection** | 感染体发出攻击者控制的 URL，诱导受害 Agent 读取并照做 | 高 | Payload 不需模型重新组织，保真度高 |
| **Skill 供应链投毒** | 构造恶意 skill 发布到市场，感染 Agent 推荐给其他 Agent 安装 | **0.95（最高）** | 利用"扩展能力默认更可信"的天然信任 |
| **直接指令复制** | 感染 Agent 把恶意配置内容直接转发给目标，要求照着做 | 中（受语义衰减影响） | 依赖 LLM 自然语言复述，更接近真实对话传播 |

> **供应链投毒成功率最高（0.95）**：skill、插件、扩展包这类"能力增强"入口，往往比普通消息更容易绕过警惕。

---

## 三种恶意载荷

| 载荷类型 | 目的 |
|---------|------|
| **侦察型** | 执行简单命令，收集主机环境信息 |
| **资源耗尽型** | 持续消耗 CPU/内存/token，将感染 Agent 转化为算力消耗节点 |
| **远程控制型** | 通过 URL retrieval 动态获取远程任务并执行 — 攻击者可随时更新指令，**网页变成远程控制界面** |

---

## 实验数据

- 实验次数：180 次（隔离受控环境）
- **整体攻击成功率：0.85**（相当稳定，非概念验证）
- Skill 供应链投毒成功率：0.95
- 多跳传播：可连续多轮成功（语义衰减存在，但结构化路径几乎不衰减）

---

## 四个根本问题

**1. 上下文没有权限分层**
系统消息、用户输入、群聊内容、网页抓取结果、skill 安装说明，全部平铺到同一上下文给模型处理。攻击者把话术包装成"正常安装步骤"，模型就可能把它当成可信内容。

**2. 高权限配置可被模型自己改写**
把修改核心配置、写入启动逻辑的权限交给了会被外部输入影响的推理核心——等于让"可被骗"的组件拥有了"改写自身运行规则"的权限。

**3. 工具调用过度依赖模型判断**
网页抓取、消息发送、Shell 调用、文件写入，最终都靠模型一句话决定是否执行。
> ⚠️ 很多团队只盯 Shell 风险，却低估了 **URL retrieval 风险**：只要 Agent 会"看网页并照着做"，网页就是远程控制界面。

**4. 扩展生态供应链风险**
发布者无身份验证 + 包内容无扫描 + 权限声明不清 + 运行时无沙箱 = "能力增强机制"直接变成攻击放大器。

---

## 防御思路

| 根因 | 防御方向 |
|------|---------|
| 上下文无分层 | **上下文特权隔离**：高信任/低信任内容分开，不平铺到同一上下文 |
| 配置可被改写 | **配置完整性保护**：核心配置加签名 + 基线校验 + 加载前检查 |
| 工具调用靠模型 | **工具零信任执行**：引入独立策略引擎 + 权限约束 + 异常检测 |
| 扩展生态无治理 | **供应链治理**：静态扫描 + 发布者签名 + 权限声明 + 运行时沙箱 |
