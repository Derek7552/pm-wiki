# AI 时代程序员三大流派

## 核心命题

在 AI 时代，程序员会不会失业，取决于你**怎么写代码**。三种范式代表三个演进阶段，适用场景完全不同。

---

## 对比总览

| 维度 | Vibe Coding | Spec Coding | Harness Engineer |
|------|------------|-------------|-----------------|
| 年份 | 2024 | 2025 | 2026 |
| 核心逻辑 | 跟着感觉走，AI 猜猜猜 | 先写清楚需求，AI 照着做 | 设计 AI 工作的环境和约束 |
| 比喻 | 理发店（说"剪短点精神点"）| 装修队（按图纸施工）| F1 赛道设计师 |
| 质量 | 抽奖 | 合同约束 | 系统工程 |
| 角色 | 用户 | 需求方 | 系统设计者 |

---

## 一、Vibe Coding（氛围编程）—— 2024

**一句话：** 给 AI 一个模糊感觉，AI 猜猜猜，能跑就上线。

**典型 prompt：**
```
"帮我加个登录功能，最好能用 Google 登录"
```

**理发店类比：**
- 你说"帮我剪短一点，看起来精神点"
- 理发师一顿操作
- 镜子一照：嗯？好像哪里不对，但又说不出来...

**适用场景判断：**

| 场景 | 建议 |
|------|------|
| 黑客马拉松演示 | 赶紧用 |
| 个人小工具 | 没问题 |
| 商业项目 | 悠着点 |
| 要命的系统 | 别作死 |

---

## 二、Spec Coding（规范编程）—— 2025

**一句话：** 先把需求写成文档，AI 照着规范执行。

**工作方式：** 在给 AI 写代码之前，先写一个功能规范文档：

```markdown
## 登录功能规范

### 验收标准
- 给定正确的邮箱+密码 → 登录成功
- 给定错误的密码 → 显示"密码错误"（不暴露用户是否存在）

### 技术方案
- 用 NextAuth.js
- 密码用 bcrypt 加密
```

**装修队类比：**
- 你把设计图拿给装修师傅
- 师傅说："好，我按图来"
- 哪里不对，按图返工，有据可查

**关键价值：** 规范文档是 AI 和人之间的"合同"，也是验收标准和复盘依据。

---

## 三、Harness Engineer（马具工程师）—— 2026

**一句话：** 不给 AI 具体任务，而是搭建 AI 工作的"跑道"——约束、工具、质量关卡。

**工作方式：**

```yaml
auth_agent:
  # 只能在这个范围内活动
  tools:
    - forbidden: [.env, secrets]
  # 跑完要过这些关卡
  quality_gates:
    - run: npm run test
    - run: npm run lint
```

**F1 赛道设计师类比：**
- 你不用亲自开车
- 你只需要设计赛道、制定规则
- 车手（AI）在你的赛道上跑，跑不出边界

**核心能力转移：** 从"怎么写代码"变成"怎么设计让 AI 可靠工作的系统"。

---

## 三段式演进逻辑

```
Vibe Coding  →  Spec Coding  →  Harness Engineer
草图（快速探索）   蓝图（规范执行）    工地（系统工程）
个人实验          团队协作             AI 规模化交付
```

**不是替代关系，而是分层关系：**
- 探索阶段用 Vibe Coding 快速验证想法
- 落地阶段用 Spec Coding 确保质量可控
- 规模化阶段用 Harness 让 AI 在约束内自主运行

---

## 对工程师的启示

- **Vibe Coding 会的人**：能快速出原型，适合探索期
- **Spec Coding 会的人**：能把需求写清楚，适合执行期
- **Harness Engineer**：能设计 AI 工作环境，是 AI 时代最稀缺的能力

> **未来不缺会 prompt 的人，缺的是会设计 Harness 的人。**
