# Agent 三大底层能力——郝建业谈记忆、安全、Harness 与生产系统落地

> 来源：FreeBuf《对话郝建业：Agent 记忆、安全以及 Harness 框架》（用户直接提供正文）；受访者：郝建业（MemoraX AI 创始人、斗象科技首席 AI 安全科学家、前华为大模型算法实验室主任 / 决策推理实验室主任、天津大学教授）
> 提炼日期：2026-05-09
> 分类：AI方法论

## TL;DR

郝建业从二十年深度强化学习研究 + 华为 6–7 年工业落地经验出发，给出一个清晰判断：**当前 Agent 走不进生产系统的根因不是基座大模型不够强，而是缺少三块底层能力——记忆、安全、Harness 框架**。其中"记忆"不是更大上下文窗口或更好 RAG 能解决的工程问题，而是 Transformer Attention 架构的固有瓶颈，必须用**专门的记忆模型 + 强化学习（Agentic RL）反向训练**——因为没有人类专家能写出"如何记住前 3 万字对话"的训练语料，记忆本质是"什么该记/什么该忘/什么时候调取"的序列决策问题，天然属于 RL 领地。安全对齐与攻防场景对 Agent 的能力要求与三短板完全重叠：基座漏洞识别能力 + Harness 工具编排 + 长链路记忆复盘。"现在很多 Agent 应用就像摆在自家后花园做试验"——demo 不叫落地，垂域生产系统才叫。

## 核心观点

### 观点一：Agent 走进垂域生产系统需要先补齐记忆、安全、Harness 三块底层能力

郝建业反复强调一个判断：**目前 Agent 在记忆、安全这些关键能力上还有明显短板，远没到统一标准、一家独大的阶段**。Agent 替代或辅助人类完成工作任务的脚手架已经被 OpenClaw（小龙虾）这类生态搭起来了，但企业不敢真把它放进核心生产系统。原因不是模型不够强，而是这三块底层能力没扎实——记忆能力不够（长期交互信息无法沉淀复用）、安全对齐能力不够（行为不可预测不可控）、Harness 不够（任务编排和工具调度能力弱）。**只有把这三块做扎实，企业和用户才敢把 Agent 部署到垂域生产系统中真正落地**。

### 观点二：记忆不是"加大窗口"问题，是必须用 Agentic RL 训练专门记忆模型的独立技术领域

主流路线是用工作流 + 外部数据库管理上下文，相当于"翻笔记"——每次要查笔记才知道之前聊了什么。郝建业反对这条路，原因有两层：第一，Transformer Attention 架构本身就有难以突破的瓶颈，**即便上下文窗口能拓展到百万级，窗口长度一旦不断拉长模型运行性能就会急剧下降，不是工程优化能解决的，是架构决定的固有问题**。第二，**记忆本质是序列决策问题——什么该记、什么该忘、什么时候调取什么信息——天然是强化学习的领地**，监督学习教不会模型"记住"，因为没有人类专家能构造出"如何记住前 3 万字对话"的训练语料。所以必须设定记忆系统的优化目标，借助 Agentic RL 反向迭代优化记忆模型参数，从模型底层实现高效记忆管理——不是翻笔记，是真正"长在模型里的记忆"。

### 观点三：攻防场景对 Agent 的能力要求与三短板精确重叠——基座 + Harness + 记忆

Claude Mythos 等自主漏洞发现能力的核心驱动有两层：基座大模型的代码漏洞感知/检测/发现能力跨过新阈值；以及智能体熟练调用各类工具、根据场景上下文自主组合工具、规划完整攻击路径的 Harness 能力。**超长链路攻防推演过程中**，智能体必须能对漫长历史交互、多轮探索的海量信息做归纳、提炼、复盘总结，做到高效自主探索——这又回到记忆问题。**攻防场景所需的能力（基座感知 + Harness 编排 + 记忆复盘 + 安全对齐）与 Agent 通用能力短板完全重叠**：两块没做好，Agent 要么"记不住"，要么"管不住"。

## 关键知识点

### 1. 三块底层能力的定义与相互关系

| 能力 | 定义 | 当前主流方案的局限 |
|------|------|------------------|
| **记忆** | 处理超长对话上下文、沉淀和复用用户长期交互信息 | 工作流 + 外部数据库 = "翻笔记"，未做到模型内生 |
| **安全（对齐）** | 从技术上约束智能体，避免损害用户利益、行为不可预测/不可控 | 提示词层 safety prompt，覆盖不到工具链层复杂攻击 |
| **Harness 框架** | 任务编排与自主调度——熟练调用各类工具，根据场景上下文自主组合工具、规划完整攻击/任务路径 | 多数 Agent 仅做工具列表罗列，无自主编排策略 |

**三者关系**：记忆是 Harness 长链路任务的支撑（让 Agent 能复盘历史），安全是 Agent 进生产系统的准入门槛，Harness 是把基座能力转化为业务结果的转换器。**三者缺一，Agent 都只能停留在 demo**。

### 2. 记忆为什么不是"更大窗口"或"RAG 优化"

**Transformer Attention 架构的固有瓶颈**：
- 上下文窗口拓展到百万级别，窗口长度一旦不断拉长模型运行性能就会急剧下降。
- 这不是工程优化能解决的，是架构决定的固有问题。

**真实场景的累积量超出基础大模型上限**：
- 用户与大模型持续深度交互产生的内容长度远超基础大模型支撑上限。
- 今天聊 5000 字、明天 5000 字、后天 5000 字——这些信息怎么沉淀？怎么被准确调取？
- 现有基础大模型做不到。

**主流"工作流 + 外部数据库"路线的局限**：
- 相当于一本笔记，每次翻笔记才知道之前聊了什么。
- 不是模型自己具备"记住"能力。

**MemoraX 主张的路线**：
- 模型内生内化、数据驱动学习。
- 专门训练具备优质记忆管理能力的模型。
- 与基础大模型结合，从模型底层实现高效记忆管理。
- "真正长在模型里的记忆"。

### 3. 为什么记忆模型的训练必须用 Agentic RL

**关键约束**：
- 传统基础大模型预训练依靠人类专家构建高质量训练语料。
- **记忆模型完全做不到这一点**——没有人能写出"如何记住前 3 万字对话"的训练语料。

**唯一可行的训练范式**：
- 设定记忆系统的优化目标（不同层面）。
- 借助强化学习反向迭代优化记忆模型参数。
- 数据驱动而非专家标注驱动。

**理论根据**：
- 记忆本质是序列决策问题——"什么该记、什么该忘、什么时候调取什么信息"。
- 这天然是强化学习的领地，监督学习教不会模型"记住"。

### 4. 攻防场景对 Agent 的三层能力要求

| 层 | 要求 | 对应短板 |
|----|------|---------|
| **基座层** | 代码漏洞感知、检测、发现能力跨过新阈值 | 基座能力，已基本满足（Claude Mythos 验证） |
| **Harness 层** | 熟练调用各类工具、根据场景上下文自主组合工具、规划完整攻击路径 | Harness 是当前明显短板 |
| **记忆层** | 超长链路攻防推演中对漫长历史交互、多轮探索的海量信息做归纳、提炼、复盘总结 | 记忆是当前明显短板 |

**结论**：攻防场景的能力要求与通用 Agent 三短板完全重叠——这意味着补齐三短板的方法论可同时服务通用 Agent 与垂域安全 Agent。

### 5. 多智能体（Multi-Agent）系统的产业落地脉络

郝建业团队过去十年把多智能体强化学习落地到的工业场景：

| 场景 | 智能体抽象 | 优化目标 |
|------|-----------|---------|
| 阿里广告推荐 | 广告主 / 用户 / 平台三方博弈 | 三方利益均衡最优 |
| 自动驾驶（华为）| 自动驾驶车 / 人类驾驶车 / 行人 | 路口交汇等复杂场景的安全与驾乘体验 |
| 无线网络优化（华为）| 不同基站作为协同智能体 | 整体优化效率 + 减少人工调优成本 |
| 供应链调度（华为）| — | 系统级全局优化 |
| 芯片设计（华为）| — | EDA 流程自动化 |

**沉淀**：一套成熟可复用的"**黑盒优化 + 白盒优化**"通用算法平台。

### 6. 强化学习的能力进化路径（郝建业亲历）

| 阶段 | 时间 | 标志 |
|------|------|------|
| 传统表格型 RL | 2013 前 | 学术界小场景验证 |
| 深度 RL 兴起 | 2013–2015 | 深度学习兴起 + AlphaGo 把表征能力 × 长周期序列决策结合，突破对传统 RL 的认知 |

**核心论断**：停留在虚拟场景里的成功不是真正的落地。

### 7. Agent 落地形态预判（1–3 年）

**短期格局**：百花齐放，市场上各式各样的"下一代智能体入口"。

**长期收敛**：会出现越来越通用的智能体入口，被认为是下一代操作系统形态。

**收敛前提**：当前 Agent 在记忆、安全等关键能力上短板明显，远没到统一标准、一家独大的阶段——只有补齐底层能力，统一入口才会出现。

**真正的落地标准**：智能体必须能真正走进各垂直领域的生产系统，而不是停留在 demo 和演示层面。

> "现在很多 Agent 应用，打个比方，就像把东西摆在自家后花园里做试验，只敢在隔离的仿真环境里跑一跑，却不敢真正放进真实业务、真实生产环境里去做操控和执行。这样的 Agent，做得再漂亮，也是温室里的花。"

## 数据与案例

### 主流 RAG 路线 vs MemoraX 路线对比

**主流（外部数据库 + 工作流）**：
- 工作机制：相当于一本笔记，每次翻笔记才知道之前聊了什么
- 局限：模型自己没有"记住"能力，外部检索引入额外延迟和召回错误
- 受限于：基础大模型的上下文窗口与 attention 性能瓶颈

**MemoraX（模型内生 + Agentic RL 训练）**：
- 工作机制：专门训练记忆模型 → 与基础大模型结合 → 从模型底层做高效记忆管理
- 优势：真正长在模型里的记忆，不依赖外部数据库
- 训练范式：设定优化目标 → Agentic RL 反向迭代 → 不依赖人类专家标注语料

### 真实场景的记忆累积压力

- 一次对话 5000 字
- 持续日交互：5000 × N 天累积
- 远超 Transformer 百万级窗口的实际可用容量（拉长后性能急剧下降）

### 攻防超长链路的记忆要求

Claude Mythos 等自主漏洞发现的能力构成：
1. 基座大模型代码漏洞感知 / 检测 / 发现能力跨过新阈值
2. Harness：熟练调用工具、根据上下文自主组合工具、规划完整攻击路径
3. 记忆：对漫长历史交互、多轮探索的海量信息做归纳、提炼、复盘总结，提升超长链路攻防任务的整体成功率

## 启发与思考

### 与个人工作的关联（云起无垠产品总监 / 安全 + AI 方向）

1. **三块底层能力清单是云起产品定位的现成框架**：把云起的安全 Agent 产品按"基座能力 + Harness 编排 + 记忆能力 + 安全对齐"四维度自检，找出短板。这与上一篇 GigaOm 雷达"前 LLM 数据层 + 智能体调查 + 集成编排"三层叠加观点同构——产品再做单点会被降维打击。
2. **攻防场景的能力要求验证了云起的方向选择**：郝建业明确指出攻防 Agent 与通用 Agent 的能力短板完全重叠——意味着云起做安全 Agent 同时也在补 Agent 通用基础设施，可双向变现（向安全客户卖产品 + 向通用 Agent 团队卖底层能力）。
3. **记忆作为独立技术领域是云起的下一个机会**：当前云起做安全 Agent 大概率仍在用主流 RAG 路线管上下文。如果 MemoraX 这条"模型内生记忆"路线被验证，云起需要尽早跟进——最差也要在 PRD 里把"记忆模型"独立成模块、不再埋在管道实现里。这与上一篇 Heuristic Learning 笔记里"前 LLM 数据层独立成章"的建议一脉相承。
4. **"后花园 demo 不叫落地"是销售对话的反向锚点**：客户对 Agent 产品的最大顾虑是"敢不敢放进生产系统"。云起销售可以反向用这个标准筛选客户痛点：不是问"您想用 Agent 做什么"，而是问"您现在哪些垂域生产环节因为安全/记忆/Harness 不够而无法引入 Agent"——把客户的"想做但不敢做"清单转化为 SOW。
5. **多智能体 + 黑白盒优化经验对漏洞挖掘场景的迁移**：郝建业团队沉淀的"黑盒优化 + 白盒优化"通用算法平台，本质上就是云起漏洞挖掘智能体场景需要的能力——黑盒模糊测试 + 白盒源码分析。可关注其学术论文与开源动作，作为云起算法演进的参考线。

- **PRD 模板升级**：在云起 Agent 类产品 PRD 中强制要求把"记忆能力 / 安全对齐 / Harness 编排"列为独立章节，禁止合并到"系统架构"里一笔带过。
- **能力自检表**：把云起当前安全 Agent 产品的记忆 / 安全 / Harness 三块能力做 1–5 分自评，形成内部基线，作为下一阶段技术演进的输入。
- **跟踪 MemoraX 与斗象科技 AI 安全产品**：郝建业双线创业（MemoraX 做记忆 / 斗象做安全），技术路线公开度可能较高，作为云起产品规划的参考标杆持续跟踪。

- **垂域落地标准内化**："demo 不算落地，进生产系统才算落地"作为云起 Agent 类产品所有客户成功案例的统一交付标准——案例必须能讲清楚客户已把 Agent 用在真实业务而非沙箱。

> "你不可能用监督学习教会模型'记住'。记忆本质上是一个序列决策问题：什么该记、什么该忘、什么时候调取什么信息——这天然是强化学习的领地。"

> "攻防场景对 Agent 的能力要求，和我说的那两个短板完全重叠：它需要记忆能力，也需要安全对齐。这两块没做好，Agent 要么'记不住'，要么'管不住'。"

> "现在很多 Agent 应用，打个比方，就像把东西摆在自家后花园里做试验，只敢在隔离的仿真环境里跑一跑，却不敢真正放进真实业务、真实生产环境里去做操控和执行。这样的 Agent，做得再漂亮，也是温室里的花。"

> "停留在虚拟场景里的成功，不是真正的落地。"
